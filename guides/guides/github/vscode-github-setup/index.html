VS Code GitHub Setup

Configuring VS Code to push and pull from multiple GitHub accounts using SSH.

Prerequisite

Complete the GitHub SSH Setup guide first. VS Code will use those SSH keys automatically.

Configure Git in VS Code

VS Code uses your system's Git installation, so your SSH config is already available. You just need to make sure Git knows your identity for commits.

Set commit identity per repo

For each repo, set the name and email that should appear on commits. Open a terminal in VS Code (Ctrl+`) and run:

For work repos:

git config user.name "skarlis3"
git config user.email "sarahlizz3@gmail.com"

For personal repos:

git config user.name "sarahlizz3"
git config user.email "sarah.karlis@pm.me"

This sets the identity only for that repo. Run these commands from within each repo folder.

Open Repositories

To work on a repo in VS Code:

  1. File > Open Folder
  2. Navigate to the repo folder (e.g., ~/Websites_work/1170)
  3. Click Open

VS Code will detect the Git repo automatically and show the branch name in the bottom-left corner.

Push and Pull

Once your repo is open and the remote URL is correctly set to use github.com-work or github.com-personal, pushing and pulling works through the Source Control panel.

Using the Source Control panel

  1. Click the Source Control icon in the left sidebar (or press Ctrl+Shift+G)
  2. Stage changes by clicking the + next to changed files
  3. Enter a commit message in the text box
  4. Click the checkmark to commit
  5. Click the three-dot menu > Push (or use the sync button)

Using the command palette

Press Ctrl+Shift+P and type:

  • Git: Push to push commits
  • Git: Pull to pull changes
  • Git: Sync to do both

Using keyboard shortcuts

There are no default shortcuts for push/pull, but you can add them in File > Preferences > Keyboard Shortcuts.

Troubleshooting

Permission denied errors

If VS Code can't push or pull:

  1. Open the integrated terminal (Ctrl+`)
  2. Check the remote URL: git remote -v
  3. If it shows https:// or plain git@github.com:, update it to use the correct alias:
git remote set-url origin git@github.com-work:skarlis3/repo.git

Replace github.com-work with github.com-personal for personal repos, and skarlis3 with sarahlizz3.

VS Code asks for GitHub login

If VS Code prompts you to sign into GitHub, you can skip it. The SSH keys handle authentication. The GitHub login is only needed if you install the optional GitHub extension for viewing PRs and issues.

Commits showing wrong author

If commits appear with the wrong name or email, you forgot to set the identity for that repo. Run the git config user.name and git config user.email commands from the Configure Git section above.

What About Settings Sync?

If you use VS Code's built-in Settings Sync (signed in with a Microsoft or GitHub account), it syncs the following across your machines:

  • Settings (your settings.json)
  • Keyboard shortcuts
  • Extensions
  • User snippets
  • UI state (panel layout, etc.)

What Settings Sync does NOT handle

Settings Sync does not sync anything related to Git authentication or identity:

  • SSH keys - These are stored on your local machine (~/.ssh) and must be generated separately on each computer. See the GitHub SSH Setup guide.
  • Git identity (user.name, user.email) - These are system-level Git settings, not VS Code settings. You need to configure them on each machine and/or in each repo.
  • Remote URLs - Each repo's remote URL is stored in that repo's .git/config file, not synced by VS Code.

Bottom line

Settings Sync is great for keeping your VS Code environment consistent (themes, extensions, keybindings), but you still need to:

  1. Set up SSH keys on each new computer (GitHub SSH Setup guide)
  2. Clone repos with the correct remote URL format (github.com-work or github.com-personal)
  3. Set git config user.name and user.email in each repo (or globally, if you only use one account)

Optional: GitHub Extension

VS Code's built-in Git support handles all push/pull operations. You do not need any extensions for basic Git workflows.

However, the "GitHub Pull Requests and Issues" extension adds extra features if you want them:

  • View and manage pull requests directly in VS Code
  • Create pull requests without leaving the editor
  • Browse and create GitHub issues
  • See inline comments from PR reviews
  • Check out PR branches with one click

Installing the extension

  1. Press Ctrl+Shift+X to open Extensions
  2. Search for "GitHub Pull Requests"
  3. Install the extension by GitHub
Single account limitation

This extension can only be signed into one GitHub account at a time. Since you have two accounts, you'll only see PRs/issues from whichever account you're signed into. This doesn't affect push/pull - SSH handles that separately.
