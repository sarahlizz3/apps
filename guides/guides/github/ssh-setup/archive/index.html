GitHub SSH Setup

Setting up SSH keys for multiple GitHub accounts on a single computer.

How It Works

Each computer needs its own SSH keys - they're stored locally and never shared between machines. This is a security feature: if one machine is compromised or you stop using it, you can revoke just that key without affecting your other computers.

When you have multiple GitHub accounts (e.g., personal and work), you create a separate key for each and use an SSH config file to tell your computer which key to use for which account.

Generate SSH Keys

1. Generate a key for your personal account

ssh-keygen -t ed25519 -C "personal" -f ~/.ssh/id_ed25519_personal

Press Enter to accept defaults when prompted. You can set a passphrase or leave it blank.

2. Generate a key for your work account

ssh-keygen -t ed25519 -C "work" -f ~/.ssh/id_ed25519_work
About the -C flag

This is just a comment/label to help you identify the key later. It doesn't affect how the key works. You can put anything there - your email, "work laptop", "linux-2025", whatever. GitHub doesn't check or care what it says.

Create SSH Config File

1. Open (or create) the config file

nano ~/.ssh/config

2. Paste this configuration

~/.ssh/config 
Host github.com-personal
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_personal

Host github.com-work
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_work

3. Save and exit

Press Ctrl+O to save, then Ctrl+X to exit nano.

How the aliases work

When a repo URL uses git@github.com-personal, it uses your personal key. When it uses git@github.com-work, it uses your work key. These aliases aren't real domains - they're just shortcuts your SSH config intercepts and routes to the real github.com with the correct key.

Add Keys to GitHub

You need to add each public key to its corresponding GitHub account.

1. Copy your personal public key

cat ~/.ssh/id_ed25519_personal.pub

Copy the entire output (the whole line starting with ssh-ed25519).

2. Add it to your personal GitHub account

  1. Go to github.com/settings/keys (while logged into your personal account)
  2. Click New SSH Key
  3. Give it a title (e.g., "Linux Desktop")
  4. Paste the key into the "Key" field
  5. Click Add SSH Key

3. Copy your work public key

cat ~/.ssh/id_ed25519_work.pub

4. Add it to your work GitHub account

Log into your work GitHub account and repeat the same steps.

Test the Connections

Verify both keys are working:

Test personal account

ssh -T git@github.com-personal

Test work account

ssh -T git@github.com-work

Each should respond with: Hi [username]! You've successfully authenticated...

First-time connection

If you see a message about "authenticity of host can't be established" with a fingerprint, type yes and press Enter. This is normal - your computer is just confirming it's really GitHub. You'll only see this once per host.

Troubleshooting

Check existing keys

See what keys you have:

ls -la ~/.ssh

Permission denied errors

  • Make sure you added the .pub (public) key to GitHub, not the private key
  • Check that the key filename in your config matches the actual file
  • Verify you added the key to the correct GitHub account

"Could not resolve hostname" errors

This usually means the SSH config file isn't set up correctly. Double-check the Host entries match what your repo URLs are using.
