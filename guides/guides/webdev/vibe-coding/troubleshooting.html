Troubleshooting

Common issues and solutions for Claude Code.

command not found: claude

The global npm bin directory isn't in your PATH.

macOS
# Find where npm installs global binaries
npm config get prefix

# Add to PATH (replace /path/to with actual path)
echo 'export PATH="/path/to/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
Pop!_OS / Linux
# Check npm prefix
npm config get prefix

# If it's /usr, the bin is /usr/bin (should already be in PATH)
# If it's ~/.npm-global, add to PATH:
echo 'export PATH="$HOME/.npm-global/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

Authentication Issues

Reset your credentials and re-authenticate:

rm -rf ~/.claude
claude

Claude Doesn't See My Files

Make sure you're running claude from your project's root directory. Claude reads files relative to where you start it.

# Check your current directory
pwd

# Navigate to project root first
cd ~/projects/my-app
claude

Changes Not Appearing in VS Code

VS Code should auto-reload files, but if it doesn't:

  • Click on the file in the explorer to refresh it
  • Or use Cmd/Ctrl + Shift + P and search "Revert File"

Large Codebases

Warning

Claude Code reads your project to understand context, which uses tokens. For very large projects, you may want to use a .claudeignore file (same syntax as .gitignore) to exclude directories like node_modules, build outputs, or large data files.

Example .claudeignore file:

node_modules/
dist/
build/
.git/
*.log
*.lock

Permission Errors on npm Install

Never use sudo with npm global installs. Instead, configure npm to use a directory in your home folder. See the Setup page for detailed instructions.

Browser Doesn't Open for OAuth

Pop!_OS / Linux

If the browser doesn't open automatically for OAuth, the terminal will display a URL. Copy and paste it into your browser manually to complete authentication.
