Usage

Basic usage, interactive mode, and one-off commands.

Basic Usage in VS Code

Opening the Terminal

macOS

Open VS Code's integrated terminal: Cmd + ` (backtick)

Pop!_OS / Linux

Open VS Code's integrated terminal: Ctrl + ` (backtick)

Starting Claude Code

  1. Navigate to your project directory (or open a folder in VS Code first)
  2. Run claude to start an interactive session
  3. Describe what you want in plain English
cd ~/projects/my-app
claude

Claude will read your project files to understand the codebase structure, then wait for your instructions.

Interactive Mode

Interactive mode is where vibe coding shines. You have a conversation with Claude about your code, and it proposes changes incrementally.

Example Session

You:
Add a dark mode toggle to the header component

Claude will find your header component, understand your styling approach (CSS, Tailwind, styled-components, etc.), and propose the implementation. You'll see exactly what files will change and what the new code looks like.

Useful Commands Within Interactive Mode

CommandDescription
/helpShow available commands
/clearClear conversation history
/compactSummarize the conversation to save context
/costShow token usage and estimated cost
/quitExit the session (or Ctrl+C)

One-Off Commands

For quick tasks, you don't need a full interactive session. Use these patterns:

Inline Task

# Claude handles the task and returns to your prompt
claude "add error handling to the API calls in src/api.js"

Print Mode (No File Changes)

# Just get an explanation without modifying anything
claude -p "explain what this function does" < src/utils/parser.js

Pipe Content In

# Analyze specific content
cat package.json | claude -p "what dependencies could be updated?"
Note

The -p flag (print mode) is useful when you want analysis or explanations without Claude making any changes to your files.
