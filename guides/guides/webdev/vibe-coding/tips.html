Tips

Effective prompting strategies and workflow tips for vibe coding.

Effective Prompting

Focus on Outcomes, Not Implementation

Good:
Users should be able to drag and drop list items to reorder them
Less Effective:
Install react-beautiful-dnd and add DragDropContext to App.js

When you describe outcomes, Claude can choose the best approach for your existing codebase and dependencies.

Iterative Refinement

Start broad, then get specific:

  1. Build a todo list component
  2. Add the ability to set due dates
  3. Color-code items that are overdue in red
  4. Add a filter to show only incomplete items

Provide Context When Helpful

With context:
We're using Zustand for state management. Add a global loading state that components can subscribe to

Ask for Explanations

Vibe coding isn't just about getting code written—it's also great for learning:

  • Explain why you chose that approach
  • What are the tradeoffs of this implementation?
  • How would this scale if we had 10x more users?

Workflow Tips

Use Git Before Major Changes

Always commit your work before asking Claude to make significant changes. This gives you an easy rollback point:

git add -A && git commit -m "checkpoint before refactor"

Keep the Terminal Visible

In VS Code, you can split your view to keep the terminal visible while looking at the files Claude is modifying:

macOS

Cmd + \ to split the editor, or drag the terminal panel to resize it.

Pop!_OS / Linux

Ctrl + \ to split the editor, or drag the terminal panel to resize it.

Use Claude for Code Review

Claude can review your own code too:

claude -p "review this code for bugs and improvements" < src/components/Form.jsx

Combine with VS Code Features

  • Source control panel — Review Claude's changes as diffs before committing
  • Problems panel — Check for any linting/type errors after changes
  • Live server — See changes in real-time as Claude modifies your code
Pro Tip

After Claude makes changes, use VS Code's source control view (Ctrl/Cmd + Shift + G) to review the diffs before committing. This helps you understand and learn from the changes.
