ID vs Class vs Style

Understanding when to use each selector type is fundamental to writing clean, maintainable CSS.

Type CSS Selector When to Use Example
id #idname Unique element (once per page); JavaScript hooks; anchor links <div id="header">
class .classname Reusable styles for multiple elements; most common for styling <div class="card">
style N/A (inline) One-off styles; quick testing; dynamically generated styles <div style="color: blue;">
Specificity

Inline style (highest) > ID > Class > Element (lowest)

Best practice: Use classes for most styling, IDs for unique elements or JS hooks, avoid inline styles except for special cases.

Pseudo-classes & Pseudo-elements

Pseudo-classes (state-based)

a:hover { }       /* When hovering over */
a:active { }      /* When clicking */
a:visited { }     /* After visiting link */
input:focus { }   /* When input is focused */
li:first-child { }   /* First child */
li:last-child { }    /* Last child */
li:nth-child(2n) { } /* Every 2nd child */

Pseudo-elements (content-based)

p::before { }       /* Insert before content */
p::after { }        /* Insert after content */
p::first-letter { } /* Style first letter */
p::first-line { }   /* Style first line */
::selection { }     /* Text selection style */

CSS Variables

CSS variables (custom properties) let you store values for reuse throughout your stylesheet:

:root {
  --primary-color: #7ac9d5;
  --spacing-unit: 1rem;
  --border-radius: 8px;
}

.button {
  background: var(--primary-color);
  padding: var(--spacing-unit);
  border-radius: var(--border-radius);
}
Why use CSS variables?

They make it easy to maintain consistent design systems, enable theming, and reduce repetition. Change one value, update everywhere!

Responsive Design

Media Queries

/* Mobile first approach */
.container {
  width: 100%;
}

/* Tablet and up */
@media (min-width: 768px) {
  .container {
    width: 750px;
  }
}

/* Desktop and up */
@media (min-width: 1024px) {
  .container {
    width: 1000px;
  }
}
