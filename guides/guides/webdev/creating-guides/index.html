Creating New Guides

How to use an AI assistant to create new pages for this documentation hub.

Quick Start Prompt

Copy this prompt and paste it at the start of a new AI session, along with your source content:

Create a new guide for my Guides Hub documentation site.

Project location: /home/skarlis/Apps/guides/

Read these files first:

  • templates/guide-instructions.md - Template and style guidelines
  • js/data.js - Current navigation structure
  • One existing guide for reference (e.g., guides/github/ssh-setup/index.html)

Then create a guide from this content: [paste your content or reference a file]

Put it in category: [App Use & Maintenance / GitHub / Linux Help / Web Dev]

What the AI Needs to Do

When creating a new guide, the AI should:

  1. Create a folder: guides/[category]/[guide-name]/
  2. Create index.html using the template structure
  3. Update navigation data in three places:
    • js/data.js
    • index.html (embedded JSON near line 33)
    • data/guides.json

File Structure

~/Apps/guides/
├── index.html                 # Homepage (has embedded nav data)
├── css/
│   ├── hub.css                # Homepage styles
│   └── guide.css              # Guide page styles
├── js/
│   ├── data.js                # Navigation data (MUST UPDATE)
│   └── nav.js                 # Sidebar navigation
├── data/
│   └── guides.json            # Navigation backup (MUST UPDATE)
├── templates/
│   └── guide-instructions.md  # Full template reference
└── guides/
    ├── apps/                  # App Use & Maintenance
    ├── github/                # GitHub guides
    ├── linux/                 # Linux Help
    └── webdev/                # Web Dev

Category Folders

Category Folder Use For
App Use & Maintenance guides/apps/ Software tools, addons, utilities
GitHub guides/github/ Git, GitHub, version control
Linux Help guides/linux/ Linux setup, commands, troubleshooting
Web Dev guides/webdev/ Web development, this site's maintenance

Style Rules

  • No emojis - Use text or SVG icons only
  • Dark mode only - Site uses dark backgrounds
  • Cool colors only - Teals, blues, purples, sage. No orange/yellow/warm tones
  • Include copy buttons - For any code blocks users might copy
  • Add quick reference table - At the top of guides with common tasks

Adding to Navigation

The new guide entry looks like this in all three data files:

{
  "id": "guide-name",
  "label": "Full Guide Name",
  "href": "guides/category/guide-name/index.html",
  "description": "Brief description for search"
}
Important

Navigation data must be updated in three files that need to stay in sync:

  • js/data.js - Used by guide page sidebars
  • index.html - Embedded in homepage
  • data/guides.json - Backup reference

HTML Template

The basic structure for a new guide page:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>PAGE TITLE | Guides Hub</title>
  <link rel="icon" type="image/png" href="../../../guide-icon-64.png">
  <link rel="stylesheet" href="../../../css/guide.css">
</head>
<body>
  <div class="guide-layout">
    <!-- Mobile sidebar toggle -->
    <button class="sidebar-toggle" aria-label="Toggle navigation">
      <svg ...></svg>
    </button>
    <div class="sidebar-overlay"></div>

    <aside class="sidebar">
      <div class="sidebar-header">
        <a href="../../../index.html" class="sidebar-logo">
          <svg ...></svg> Guides Hub
        </a>
      </div>
      <nav class="sidebar-nav"></nav>
    </aside>

    <main class="guide-main">
      <article class="guide-content">
        <nav class="breadcrumbs">
          <a href="../../../index.html">Home</a>
          <span class="breadcrumb-separator">/</span>
          <span class="breadcrumb-current">Guide Title</span>
        </nav>

        <h1 class="guide-title">Guide Title</h1>
        <p class="guide-description">Brief description</p>

        <!-- Your content here -->

      </article>
    </main>
  </div>

  <script src="../../../js/data.js"></script>
  <script src="../../../js/nav.js"></script>
</body>
</html>
Tip

For the complete template with all SVG icons and copy button code, have the AI read an existing guide like guides/github/ssh-setup/index.html as a reference.
