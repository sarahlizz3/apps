Site Sync

A GUI tool for pulling GitHub repos to your local machine. Works on Mac and Linux.

What It Does

  • Pull entire sites or just specific folders
  • Pull only root files (index.html, js, css) for quick edits
  • Different default save locations per computer
  • Add/remove sites through the settings panel

Prerequisites

You need Python 3, Git, and Tkinter (Python's GUI library). These are usually pre-installed on Mac and Linux.

Check if you have them

Run each command to verify installation:

python3 --version
git --version
python3 -c "import tkinter; print('tkinter OK')"

If tkinter is missing

Mac:

brew install python-tk

Linux (Pop!_OS / Ubuntu / Debian):

sudo apt install python3-tk

Setup

1. Download the files

Put these files somewhere permanent on your computer:

  • site_sync.py (the main app)
  • config.json (your settings)

Suggested locations:

  • Mac: ~/Applications/SiteSync/
  • Linux: ~/Apps/SiteSync/

2. Run the app

Navigate to your SiteSync folder and run:

cd ~/Apps/SiteSync && python3 site_sync.py
Note

On Mac, replace ~/Apps/SiteSync with ~/Applications/SiteSync or wherever you saved the files.

3. Configure your sites

Click Settings in the app, or edit config.json directly.

For each site, you need to set:

  • Repo URL: Your GitHub repo URL (e.g., https://github.com/yourusername/1190.git)
  • Default Path: Where to save files (leave blank to use global default)
  • Folders: Subfolders you might want to pull individually

Example config.json for Mac

config.json 
{
  "sites": {
    "1190": {
      "repo": "https://github.com/skarlis/1190.git",
      "default_path": "",
      "folders": ["assignments", "readings", "resources", "policies", "calendar"]
    }
  },
  "root_files": ["index.html", "*.js", "*.css", "*.json"],
  "global_default_path": "/Users/skarlis/Library/Mobile Documents/com~apple~CloudDocs/4_Work/1_Web"
}

Example for Linux

Change the global_default_path to your preferred location:

config.json (partial) 
{
  "global_default_path": "/home/yourusername/Sites"
}

Making It Clickable (Optional)

Mac: Create an app icon

  1. Open Automator (search for it in Spotlight)
  2. Choose Application
  3. Search for "Run Shell Script" and drag it over
  4. Paste this (adjust the path if needed):
cd ~/Applications/SiteSync && python3 site_sync.py
  1. Save as "Site Sync" to your Applications folder
  2. (Optional) Change the icon: Right-click > Get Info > drag an image onto the icon

Linux: Create a desktop shortcut

Create the desktop entry file:

nano ~/.local/share/applications/site-sync.desktop

Paste this content (adjust the path to match your setup):

site-sync.desktop 
[Desktop Entry]
Name=Site Sync
Comment=Pull GitHub repos
Exec=python3 /home/skarlis/Apps/SiteSync/site_sync.py
Icon=folder-download
Terminal=false
Type=Application
Categories=

Make it executable:

chmod +x ~/.local/share/applications/site-sync.desktop

Usage

  1. Select a site from the dropdown
  2. Choose what to pull:
    • Entire site - everything
    • Root files only - just index.html, .js, .css files
    • Specific folder - pick from the dropdown
  3. Verify the save location (or browse to change it)
  4. Click Pull

The status box will show progress and any errors.

Adding a New Site

Option 1: Through the app

  1. Click Settings
  2. Click Add Site
  3. Fill in the name, repo URL, default path, and folders
  4. Click Update Site
  5. Click Save

Option 2: Edit config.json directly

Add a new entry to the "sites" object:

config.json (partial) 
"new-class": {
  "repo": "https://github.com/yourusername/new-class.git",
  "default_path": "",
  "folders": ["assignments", "readings"]
}

Troubleshooting

"Git is not installed"

Mac:

xcode-select --install

Or, if you have Homebrew:

brew install git

Linux:

sudo apt install git

"Pull failed" errors

  • Make sure the repo URL is correct
  • Make sure you have access to the repo (try git clone <url> manually)
  • Check your internet connection

App won't open

  • Make sure Python 3 and tkinter are installed
  • Try running from terminal to see error messages:
python3 /path/to/site_sync.py

Files saving to wrong location

  • Check the save path before clicking Pull
  • Update the default path in Settings

Notes

  • The app creates/updates a .git folder in your save location - this is normal
  • If you edit files locally and also on another computer, you may get merge conflicts when pulling. The app will show an error if this happens.
  • Each computer has its own config.json, so you can have different default paths on Mac vs Linux
