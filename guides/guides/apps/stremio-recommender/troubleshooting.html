Troubleshooting

Common issues and how to fix them

Common Issues

Addon Not Showing in Stremio

  • Make sure the addon is running (npm start in the addon folder)
  • Check if the addon is installed: Stremio > Settings > Addons
  • Try reinstalling: remove the addon and add http://localhost:7000/manifest.json

No Recommendations Appearing

  • Open the dashboard and click "Generate" to create recommendations
  • Check if your API keys are configured correctly
  • Look at the terminal running the addon for error messages

Dashboard Won't Load

  • Verify the addon is running (check terminal output)
  • Make sure nothing else is using port 7000
  • Try http://127.0.0.1:7000 instead of localhost

Generation Fails or Times Out

  • Check your Claude API key is valid and has credits
  • Check your TMDb API key is valid
  • Try generating again - transient errors can occur

Running as a Background Service

To have the addon start automatically and run in the background, use pm2:

Install pm2

npm install -g pm2

Start the Addon with pm2

cd ~/Apps/stremio-recommender
pm2 start npm --name "stremio-recommender" -- start

Save pm2 Configuration

pm2 save

Set pm2 to Start on Boot

pm2 startup

This will output a command to run. Copy and run that command.

Useful pm2 Commands

Command What It Does
pm2 status Show all running processes
pm2 logs stremio-recommender View addon logs
pm2 restart stremio-recommender Restart the addon
pm2 stop stremio-recommender Stop the addon
pm2 delete stremio-recommender Remove from pm2

File Reference

Key files in the addon folder (~/Apps/stremio-recommender/):

File Purpose
taste-profile.json Your preferences for recommendations
data/recommendations.json Current recommendations list
data/dismissed.json Titles you've dismissed
.env API keys (Claude, TMDb)
config.json Addon configuration
Backup Tip

Back up taste-profile.json and data/dismissed.json - these contain your personalization data.

Getting Help

If you're still having issues:

  1. Check the terminal output for error messages
  2. Look at pm2 logs if running as a service
  3. Verify all API keys are correctly set in .env
