Writing Effective Prompts

The quality of your prompts directly affects the quality of Claude's output.

Key Principles

1. Be Clear and Specific

Don't describe the symptomâ€”describe the problem and your constraints.

Vague:
"The app is slow"
Clear:
"Typing in the search box is sluggish when there are more than 200 results. Profile the SearchResults component and find what's blocking the main thread."

2. Be Explicit About What You Want

If something matters to you, say it.

Implicit:
"Add a user registration endpoint"
Explicit:
"Add a user registration endpoint with email validation, password hashing using bcrypt, and rate limiting. Include unit tests."

3. Give Context and Constraints

Tell Claude what it's working with and what boundaries exist.

  • Reference specific files with @
  • Mention relevant dependencies or APIs
  • Explain why something needs to work a certain way

4. One Mode Per Prompt

Don't mix different types of work in one prompt:

  • Build mode: "Create X" or "Implement Y"
  • Debug mode: "Here's the error, here's the log, find the bug"
  • Learn mode: "Explain how X works"
  • Critique mode: "Review this code for security issues"

5. Tell Claude How to Solve, Not Just What's Broken

If you have domain knowledge about the fix, share it.

What's broken:
"Users are getting logged out too fast"
How to fix:
"Users are being logged out after 5 minutes. The session timeout in @config/auth.ts should be 3600 seconds, not 300. Find where this is set and fix it."

6. Request Action, Not Suggestions

Claude Code can make changes directly. Ask for what you want done.

Passive:
"Can you suggest some changes to improve this function?"
Active:
"Refactor this function to improve its performance. Then run the tests."

Prompt Comparison Examples

Less Effective More Effective
"Fix the bug" "The login form throws 'Cannot read property email of undefined' when submitted empty. Check @src/components/LoginForm.tsx for missing null checks."
"Make the code better" "Refactor @src/api/users.ts to use async/await instead of .then() chains. Add proper error handling that returns appropriate HTTP status codes."
"Add tests" "Add unit tests for validateEmail and validatePassword in @src/lib/validation.ts. Test valid inputs, edge cases (empty strings, special characters), and invalid formats. Use Jest."
"Why is this slow?" "The UserList component re-renders on every keystroke in the search box. Profile render performance and implement memoization where appropriate."
"Help with git" "I have merge conflicts in @src/config.ts between main and feature-branch. Show me the conflicts, explain what each version does, and help me choose the right resolution."
"Review my code" "Review @src/auth/session.ts for security vulnerabilities, particularly around token handling, session expiration, and CSRF protection."

The Pattern

Effective prompts typically include:

  • What needs to happen (specific action)
  • Where it should happen (file paths with @)
  • Why (context about the problem or goal)
  • How (constraints, patterns to follow, tools to use)
  • Verification (how to confirm it worked)
