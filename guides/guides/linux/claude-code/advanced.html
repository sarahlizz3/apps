Advanced Usage

Headless mode, custom commands, session management, and more.

Headless Mode

Run Claude Code without an interactive session for scripts and automation:

# One-off query
claude -p "Explain what this project does"

# Get JSON output for scripts
claude -p "List all API endpoints" --output-format json

# Streaming JSON for real-time processing
claude -p "Analyze this log file" --output-format stream-json

Session Management

# Continue last session
claude -c

# Resume a specific session
claude --resume

# List recent sessions
claude --list

Custom Slash Commands

Create reusable command templates by adding markdown files to .claude/commands/:

Example: PR Command

Create .claude/commands/pr.md:

---
description: Prepare and create a pull request
---

Prepare a pull request for the current changes:
1. Run linting and fix any issues
2. Run tests and ensure they pass
3. Stage all changes
4. Write a descriptive commit message following conventional commits
5. Push the branch
6. Create a PR with gh cli including:
   - Clear title summarizing the change
   - Description of what changed and why
   - Any testing notes

Then use it with:

> /pr

Commands with Arguments

Create .claude/commands/issue.md:

---
argument-hint: [issue-number]
description: Fix a GitHub issue
---

Analyze and fix GitHub issue #$ARGUMENTS:
1. Use gh issue view to get issue details
2. Search the codebase for relevant files
3. Implement the fix
4. Run tests
5. Create a commit with "Fixes #$ARGUMENTS" in the message

Use with:

> /issue 42

Model Selection

Claude Code supports different models:

# Use Claude Sonnet (faster, cheaper)
export ANTHROPIC_MODEL="claude-sonnet-4-5-20250929"

# Use Claude Opus (most capable)
export ANTHROPIC_MODEL="claude-opus-4-5-20251101"

Parallel Sessions

Run multiple Claude instances for different tasks:

# Terminal 1: Working on feature
cd project && claude

# Terminal 2: Running tests and review
cd project && claude -p "Run all tests and explain any failures"

Git Workflow Integration

Claude Code excels at git operations. Many developers use it for 90%+ of their git work:

> What changed since the last release?
> Write a commit message for my staged changes
> Help me resolve merge conflicts in @package.json
> Create a PR for this branch with a description of changes
> Who last modified the authentication module and why?

Using External CLIs

Tell Claude to use CLI tools you have installed:

> Use gh to list open issues labeled "bug"
> Use aws cli to check the S3 bucket permissions
> Use docker to show running containers
Tip

If you frequently use external tools, document them in your project's CLAUDE.md so Claude knows they're available.
