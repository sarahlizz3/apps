Project Memory

Using CLAUDE.md for persistent project context across sessions.

What is CLAUDE.md?

CLAUDE.md is a special file that Claude Code automatically loads at the start of every session. It serves as your project's "memory"—storing conventions, decisions, and context that persist across sessions.

Memory Hierarchy

Claude Code loads CLAUDE.md files in this order (most specific wins):

  1. Global

    ~/.claude/CLAUDE.md — applies to all projects

  2. Project

    ./CLAUDE.md — checked into version control

  3. Project Local

    ./CLAUDE.local.md — gitignored, personal settings

  4. Subdirectory

    ./tests/CLAUDE.md — context specific to that folder

Creating a CLAUDE.md

You can ask Claude to create one:

> /init

Or create it manually. Here's a template:

# CLAUDE.md
# Project guidance for Claude Code

## Project Overview
Brief description of what this project does.

## Tech Stack
- Language: TypeScript
- Framework: React
- Testing: Jest + React Testing Library

## Bash Commands
- `npm run build` - Build the project
- `npm run dev` - Start development server
- `npm run test` - Run tests
- `npm run lint` - Run linter

## Code Style
- Use ES modules (import/export), not CommonJS (require)
- Destructure imports when possible
- Use TypeScript strict mode
- Prefer functional components with hooks

## Workflow
- Run typecheck after making code changes
- Write tests for new functionality
- Follow existing patterns in the codebase

## Project Structure
- `/src/components` - React components
- `/src/lib` - Utility functions
- `/src/api` - API routes and handlers
- `/tests` - Test files mirror src structure

Quick Memory (# Prefix)

Add quick notes to memory during a session:

> # Always use async/await, never callbacks
> # The auth module is in /src/lib/auth.ts

Best Practices

  • Keep it concise: This file is loaded with every request. Bloated CLAUDE.md wastes tokens.
  • Be specific: Include actual commands, file paths, and conventions.
  • Refactor regularly: Remove outdated information and consolidate repeated patterns.
  • Use subdirectory CLAUDE.md files: Put test-specific guidance in /tests/CLAUDE.md.
Tip

Use /memory to quickly edit your project's CLAUDE.md file without leaving your session.
