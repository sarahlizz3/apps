Troubleshooting

Common issues and solutions for Claude Code on Linux.

Diagnostic Command

claude doctor

This auto-detects most configuration issues and suggests fixes.

Common Issues

"Command not found: claude"

The binary isn't in your PATH. Add it:

# For native installation
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# For npm installation
echo 'export PATH="$(npm config get prefix)/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

Permission Errors with npm

Never use sudo with npm. Instead, configure a user directory:

mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

Authentication Failures

  1. Check your API key: echo $ANTHROPIC_API_KEY
  2. Verify it starts with sk-ant-
  3. Check billing status at console.anthropic.com
  4. Try re-authenticating: close Claude and run claude again

Segmentation Fault

Usually means mixed or outdated installation. Clean reinstall:

# Remove old installation
npm uninstall -g @anthropic-ai/claude-code
rm -rf ~/.claude ~/.npm/_cacache

# Clean install with native method
curl -fsSL https://claude.ai/install.sh | bash

Context Window Getting Full

Long conversations consume the context window. Manage it:

> /context    # Check usage
> /compact    # Compress while preserving key info
> /clear      # Start fresh if needed

Claude Keeps Asking for Permission

Configure default permissions to reduce prompts:

> /permissions

Or add trusted operations to your settings.

Getting Help

  • In-app: /bug to report issues
  • Documentation: claude.ai/code
  • GitHub: github.com/anthropics/claude-code

Uninstalling

# Remove binary
rm -rf ~/.local/bin/claude
rm -rf ~/.local/share/claude
rm -rf ~/.claude/bin

# Remove configuration (optional - deletes all settings)
rm -rf ~/.claude
rm ~/.claude.json

# Remove project settings (from project directory)
rm -rf .claude
rm .mcp.json
