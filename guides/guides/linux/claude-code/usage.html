Usage

Basic usage, keyboard shortcuts, and slash commands reference.

Starting a Session

# Navigate to your project directory first
cd /path/to/your/project

# Start Claude Code
claude

Claude automatically reads your project structure and understands the codebase context.

Basic Interaction

Once inside a session, just type naturally:

> What does this project do?
> Find the authentication logic and explain how it works
> Add a new endpoint for user registration

Referencing Files

Use @ to reference specific files:

> Review @src/auth/login.ts for security issues
> Add tests for @lib/utils.js

Running Bash Commands

Prefix with ! for direct bash execution:

> !git status
> !npm test

Keyboard Shortcuts

ShortcutAction
Ctrl+CCancel current operation
Ctrl+RSearch command history
TabToggle thinking mode visibility
Shift+TabToggle between modes
Esc EscRewind/undo last action
Up ArrowNavigate through past prompts
Ctrl+DExit session

Slash Commands Reference

Essential Commands

CommandDescription
/helpShow available commands and help
/clearClear conversation history (start fresh)
/compactCompress context to save tokens while preserving important info
/contextView current context window usage
/costShow token usage and estimated costs
/statusView current session status
/exitExit the session

Memory & Configuration

CommandDescription
/memoryEdit the CLAUDE.md file (project memory)
/permissionsManage tool permissions and allowed domains
/configAccess configuration settings
/initInitialize a new CLAUDE.md for your project

Session Management

CommandDescription
/resumeResume a previous session
/renameGive current session a memorable name
/reviewRequest a code review
/bugReport an issue with Claude Code
Pro Tip: Use /clear Often

Every time you start something new, clear the chat. Old history eats tokens and can confuse context. Use /clear liberally.
