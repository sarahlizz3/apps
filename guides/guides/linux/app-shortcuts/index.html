App Menu Shortcuts for Local HTML Files

Create clickable launcher shortcuts for local HTML applications on Pop!_OS with COSMIC desktop.

Contents
  • Overview
  • Quick Start
  • Detailed Steps
  • Field Reference
  • Browser Options

Overview

This guide shows you how to create a clickable shortcut in your applications menu that opens a local HTML file in your default browser. The shortcut will appear in the COSMIC launcher and applications view just like any other app.

We'll create a .desktop file, which is the standard way Linux desktop environments handle application shortcuts.

Quick Start

Create a file at ~/.local/share/applications/my-app.desktop with this content:

[Desktop Entry]
Version=1.0
Type=Application
Name=My Local App
Comment=Opens my local HTML application
Exec=xdg-open /home/YOUR_USERNAME/path/to/your/file.html
Icon=text-html
Categories=Utility;
Terminal=false

Replace YOUR_USERNAME and the path with your actual values. Save the file, and the app should appear in your launcher.

Detailed Steps

Step 1: Create the Applications Directory (if needed)

Open a terminal and run:

mkdir -p ~/.local/share/applications

Step 2: Create the .desktop File

You can use any text editor. Here's how with nano:

nano ~/.local/share/applications/my-local-app.desktop

Step 3: Add the Desktop Entry Content

Paste the following, customizing the values:

~/.local/share/applications/my-local-app.desktop 
[Desktop Entry]
Version=1.0
Type=Application
Name=My Dashboard
GenericName=Local Web App
Comment=My personal dashboard application
Exec=xdg-open /home/sarah/Projects/dashboard/index.html
Icon=text-html
Categories=Utility;
Terminal=false
StartupNotify=true

Step 4: Save and Exit

In nano: Press Ctrl+O to save, then Ctrl+X to exit.

Step 5: Make it Executable (Optional but Recommended)

chmod +x ~/.local/share/applications/my-local-app.desktop

Step 6: Verify

Press the Super key to open the COSMIC launcher and search for your app name. It should appear in the results.

Field Reference

Field Required Description
[Desktop Entry] Yes Header that identifies this as a desktop entry file
Version No Desktop entry spec version (use 1.0)
Type Yes Must be Application for launchable apps
Name Yes The name shown in the menu and launcher
GenericName No Generic description (e.g., "Web Browser")
Comment No Tooltip text shown on hover
Exec Yes Command to run. Use xdg-open for HTML files
Icon No Icon name or full path to icon file
Categories No Menu categories (semicolon-separated, ends with semicolon)
Terminal No Set to false for GUI apps
StartupNotify No Show loading indicator while app starts

Browser Options

If you want to force a specific browser instead of the system default:

# For Firefox
Exec=firefox /home/sarah/Projects/dashboard/index.html

# For Chromium/Chrome (opens as app-like window)
Exec=chromium --app=file:///home/sarah/Projects/dashboard/index.html

# For Google Chrome
Exec=google-chrome --app=file:///home/sarah/Projects/dashboard/index.html
App Mode

The --app= flag opens the page without browser chrome (no address bar, tabs, etc.), making it feel more like a native application.
