Troubleshooting

Common issues and solutions for desktop entry shortcuts.

Quick Fixes

App doesn't appear in launcher

  • Verify the file is in ~/.local/share/applications/
  • Check the file ends with .desktop
  • Ensure no syntax errors (no spaces around = signs)
  • Try logging out and back in
  • Refresh the desktop database:
update-desktop-database ~/.local/share/applications/

Validate your .desktop file

Check for syntax errors:

desktop-file-validate ~/.local/share/applications/my-app.desktop

Icon not showing

  • Check the icon name is spelled correctly
  • For custom icons, verify the path exists:
ls -la /path/to/icon.png
  • Try using a full path instead of an icon name
  • Update the icon cache:
gtk-update-icon-cache ~/.local/share/icons/hicolor/

App won't launch

  • Test the Exec command in terminal first
  • Check the HTML file path is correct and accessible
  • Try using the file:// prefix:
Exec=xdg-open file:///home/user/app/index.html

Removing a shortcut

rm ~/.local/share/applications/my-app.desktop

Complete Examples

Simple Dashboard

~/.local/share/applications/my-dashboard.desktop 
[Desktop Entry]
Version=1.0
Type=Application
Name=My Dashboard
Comment=Personal productivity dashboard
Exec=xdg-open /home/sarah/Projects/dashboard/index.html
Icon=go-home
Categories=Utility;
Terminal=false

Documentation Site (Chrome App Mode)

~/.local/share/applications/project-docs.desktop 
[Desktop Entry]
Version=1.0
Type=Application
Name=Project Docs
GenericName=Documentation
Comment=Local documentation for my project
Exec=google-chrome --app=file:///home/sarah/docs/index.html
Icon=help-browser
Categories=Development;Documentation;
Terminal=false
StartupNotify=true

With Custom Icon

~/.local/share/applications/flow-app.desktop 
[Desktop Entry]
Version=1.0
Type=Application
Name=Flow App
Comment=Productivity tracker
Exec=firefox /home/sarah/apps/flow/index.html
Icon=/home/sarah/apps/flow/icon.png
Categories=Office;ProjectManagement;
Terminal=false
