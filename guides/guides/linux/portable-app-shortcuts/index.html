Portable App Shortcuts

Add AppImages, tarballs, and portable binaries to your COSMIC applications menu.

In This Guide
  • Overview - What are portable apps?
  • AppImages - Self-contained application files
  • Tarballs - Extracted archive programs
  • Icons - Standard names and custom icons
  • Troubleshooting - Common issues and examples

Overview

Many Linux applications are distributed as portable programs that don't require installation:

  • AppImages - Single executable files (e.g., MyApp.AppImage)
  • Tarballs - Extracted archives with a binary (e.g., ./myapp/bin/myapp)
  • Portable binaries - Standalone executables downloaded directly

This guide shows you how to add these to your COSMIC applications menu by creating a .desktop file.

Quick Start

Create a file at ~/.local/share/applications/my-program.desktop:

[Desktop Entry]
Version=1.0
Type=Application
Name=My Program
Comment=Description of what the program does
Exec=/home/YOUR_USERNAME/Applications/MyProgram.AppImage
Icon=application-x-executable
Categories=Utility;
Terminal=false

Replace the path in Exec with the actual path to your program. Make sure the program file is executable:

chmod +x /home/YOUR_USERNAME/Applications/MyProgram.AppImage

Detailed Steps

Step 1: Organize Your Portable Apps

Create a dedicated folder for portable applications:

mkdir -p ~/Applications

Move or copy your downloaded program there.

Step 2: Make the Program Executable

chmod +x ~/Applications/YourProgram

Step 3: Test the Program

Run it from terminal to make sure it works:

~/Applications/YourProgram

Step 4: Create the .desktop File

nano ~/.local/share/applications/your-program.desktop

Step 5: Add the Desktop Entry Content

~/.local/share/applications/your-program.desktop 
[Desktop Entry]
Version=1.0
Type=Application
Name=Your Program
GenericName=Program Type
Comment=What this program does
Exec=/home/sarah/Applications/YourProgram %U
Icon=application-x-executable
Categories=Utility;
Terminal=false
StartupNotify=true
StartupWMClass=your-program

Step 6: Save and Make Executable

chmod +x ~/.local/share/applications/your-program.desktop

Field Reference

Field Required Description
Type=Application Yes Identifies this as a launchable application
Name Yes Display name in the menu
Exec Yes Full path to the executable. Use %U to accept file arguments
Icon No Icon name or path (see Icons)
Categories No Menu categories (see below)
Terminal No false for GUI apps, true for CLI tools
StartupWMClass No Window class name (helps with taskbar grouping)
Path No Working directory to run the program from
