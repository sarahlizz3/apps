Common issues, solutions, and complete working examples.

Program won't launch from menu

  1. Test the Exec command directly in terminal:
/home/user/Applications/MyProgram.AppImage
  1. Check if it's executable:
chmod +x /path/to/program
  1. For AppImages, try adding --no-sandbox flag if it fails
  2. Check for missing libraries:
ldd /path/to/program

Shortcut doesn't appear in launcher

  1. Verify file location:
ls ~/.local/share/applications/
  1. Check file extension is .desktop
  2. Validate syntax:
desktop-file-validate ~/.local/share/applications/myapp.desktop
  1. Refresh database:
update-desktop-database ~/.local/share/applications/
  1. Try logging out and back in

Icon not displaying

  1. Check the icon path exists:
ls -la /path/to/icon.png
  1. Try using a full path instead of icon name
  2. Try a standard icon name like application-x-executable
  3. Update icon cache:
gtk-update-icon-cache ~/.local/share/icons/hicolor/

Electron/Chrome apps show wrong icon in taskbar

Add the StartupWMClass field. To find the correct value:

# Launch the app, then run:
xprop WM_CLASS
# Click on the app window
# Use the second value in the output

App opens in terminal unexpectedly

Make sure you have:

Terminal=false

Removing a shortcut

rm ~/.local/share/applications/myapp.desktop

Complete Examples

CLI Tool (Runs in Terminal)

~/.local/share/applications/htop.desktop 
[Desktop Entry]
Version=1.0
Type=Application
Name=htop
GenericName=Process Viewer
Comment=Interactive process viewer
Exec=/home/sarah/Applications/htop
Icon=utilities-system-monitor
Categories=System;Monitor;
Terminal=true
Terminal Apps

For CLI tools that need a terminal, set Terminal=true. The app will open in your default terminal emulator.

Reference

This guide follows the freedesktop.org Desktop Entry Specification.
