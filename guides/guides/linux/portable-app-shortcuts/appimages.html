Setting Up an AppImage

# Move to Applications folder
mv ~/Downloads/SomeApp-x86_64.AppImage ~/Applications/

# Make executable
chmod +x ~/Applications/SomeApp-x86_64.AppImage

# Optional: Rename for cleaner paths
mv ~/Applications/SomeApp-x86_64.AppImage ~/Applications/SomeApp.AppImage

Example Desktop Entries

Obsidian

~/.local/share/applications/obsidian.desktop 
[Desktop Entry]
Version=1.0
Type=Application
Name=Obsidian
GenericName=Markdown Editor
Comment=Knowledge base and note-taking
Exec=/home/sarah/Applications/Obsidian.AppImage %U
Icon=obsidian
Categories=Office;TextEditor;
Terminal=false
StartupWMClass=obsidian
MimeType=x-scheme-handler/obsidian;

Cursor IDE

~/.local/share/applications/cursor.desktop 
[Desktop Entry]
Version=1.0
Type=Application
Name=Cursor
GenericName=Code Editor
Comment=AI-powered code editor
Exec=/home/sarah/Applications/Cursor.AppImage --no-sandbox %F
Icon=code
Categories=Development;IDE;TextEditor;
Terminal=false
StartupWMClass=Cursor
Electron Apps

Some Electron-based AppImages (like Cursor) may need the --no-sandbox flag to run properly.

Blender

Extracting Icons from AppImages

Many AppImages contain their own icons. You can extract them:

# Extract AppImage contents
cd ~/Applications
./SomeApp.AppImage --appimage-extract

# Find icons
find squashfs-root -name "*.png" -o -name "*.svg" | head -10

# Copy the icon you want
cp squashfs-root/usr/share/icons/hicolor/256x256/apps/someapp.png ~/icons/

# Clean up
rm -rf squashfs-root
Common Icon Locations in AppImages

After extracting, check these paths:

  • squashfs-root/*.png or *.svg
  • squashfs-root/usr/share/icons/
  • squashfs-root/.DirIcon
